### **元件基本範例及結構**

元件使用的基本要點

- 元件需要在 createApp 後，mount 前進行定義
- 元件需指定一個名稱
- 元件結構與最外層的根元件結構無異（除了增加 Template 的片段）
- 元件另有 prop, emits 等資料傳遞及事件傳遞

### **不同元件的註冊方式**

### **全域註冊**

**此 createApp 下，任何子元件都可運用，在中小型專案、一般頁面開發很方便**

直接串接在 create.App 後方，使用樣本字面詞來新增上，`.component('alert',` 此地方的 alert ，為此 template 的命名，只需要在 html 上新增 `<alert></alert>` 就會呈現完整的 template 樣式以及資料內容

假使有多個元件需要新增，可以直接在 app.mount 上做新增，例如下方的 alert2

```jsx
const app = Vue.createApp({
	data(){
		return {
			text: '測試',
		}
	},
}).component('alert', {
		data(){
			return {
				text: '內部文字'
			};
		},
		template: `<div class="alert alert-primary" role="alert">
								{{ text }}
							</div>`
	}),

app.component('alert2',{
	data() {
		return {
			text: '元件2'
		}
	},
	template: `<div class="alert alert-primary" role="alert">
								{{ text }}
							</div>`
	})

	app.mount('#app')
```

### **區域註冊**

限制在特定元件下才可使用，在 Vue Cli 中很常使用此方法（便於管理）

可以在 Vue Cli 內快速查詢到元件位置，方便未來做修改更新

在根元件前方新增變數，最後需要在根目錄回傳此變數，在 data 後方新增上物件 `components:{}`並且在物件內新增上此變數名稱，再到 html 上新增上此變數名稱，即可顯示 

```jsx
const alert3 = {
    data() {
      return {
        text: '元件 3'
      };
    },
    template: `<div class="alert alert-primary" role="alert">
      {{ text }}
    </div>`
  }

const app = Vue.createApp({
    data() {
      return {

      };
    },
    components:{
      alert3
    }
  })

app.mount('#app')
```

### **模組化**

同屬於區域註冊，Vue Cli 中的實戰運用技巧

新增一支 js 檔案，命名為 component-alert4 來做撰寫

```jsx
// component-alert4.js

export default {
  data() {
    return {
      text: "外部匯入的元件",
    };
  },
  template: `<div class="alert alert-primary" role="alert">
        {{ text }}
    </div>`,
};
```

回到主件上， 將 component-alert4 使用 `import` 至前方，並且透過區域元件方式，新增在裡頭

```jsx
// main.js
import alert4 from './component-alert4.js'

  const app = Vue.createApp({
    data() {
      return {

      };
    },
    components:{
      alert4
    }
  })
```

最後再回到 html 內，新增上所命名的區塊 `<alert4></alert4>` 就會渲染出來了

```html
<div id="app">
	<alert4></alert4>
<div>
```
