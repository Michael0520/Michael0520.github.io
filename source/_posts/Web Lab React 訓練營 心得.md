# Web Lab React 訓練營 心得

## 簡介
總長為 12週，課程訓練相當緊湊，學習到的內容都是目前最新的前端資訊

課程開始之前
## 為什麼選擇這個課程

當初會知道 KK 和 React 訓練營計畫，是因為平時有在追蹤些前端社群的資訊，
期許自己未來有天能跨出去參加，並且多認識些前端圈的朋友，
此課程有個試聽會，如果可以聽得懂課程才再來參加，覺得 KK 把課程大綱寫得很簡明扼要，前端語言以及工具安排的循序漸進，
透過日常情境的方式來教導，讓人很能感同身受也印象深刻，教課風格挺有趣的。（這點在之後的課程中也會體驗到。）

> 如果能將課綱上所提的內容融會貫通，應該就真的如計畫簡章所宣稱的，能在結業後找到工程師的工作，
這點相當吸引到我，希望能夠藉此減少許多不必要的歪路，直達我的需求核心。

我是在上份工作在新創公司擔任業務時，才認識到何謂程式語言，那時公司主要的產品是製作聊天機器人，
藉由機器人聊天機制來快速，讓使用者獲得他想得到的資訊，
過層中因為需要教導客戶端後台，因為後台是使用 wordpress 構造，而學習到了基礎的 html , css ,
也對程式語言產生了興趣，因而下定決心全職來學習。

## 規則吸引我的地方

1. 直播講情境大綱 + 線上課程講細節
2. 「任何時候」都可以問問題 (這實在太重要了！)
3. 有討論區來讓學員們，共同學習的方式來討論問題解法(喜歡這種方式，感覺比較不孤單)
4. 淘汰機制，讓我能夠督促自己的學習進度，如果真的被淘汰了，也能夠清楚自己不足的部分未來加以補足！
5. 期末專案，能夠有個場地來展示自己的作品，這點像是以前在做大學專題的感覺，相當熟悉
6. 成果分享，未來能夠在這留下來，持續為這個社群推進，推廣更多前端技術，是我很憧憬的一件事！

備註：

> 自學能力是工程師的一大條件，課程內不時會放入些問題，讓我們去尋找解答，
對初學者來說是一個 debug。雖然知道自己一定是哪裡寫錯了，
但能不能找到解決辦法，真的是一大難事啊，
這次的訓練營讓我學習到找資源的重要性！

## 對課程的期待
有效率的學習以及有挑戰性的解決任務到達最後一關，獲得屬於自己的專題發表。

## 課程開始之後：關於學習歷程與轉變


### 前期：（實際時間：前四週課程）
還以為前三週的學習，會是挺輕鬆的，殊不知前三週的三大基礎資訊量相當龐大，
HTML , CSS , JavaScript 學習的將當緊湊，需要實作許多小專案才能加速理解，
尤其是 JavaScript，再 pormise 以及 同步非同步中，花了相當多的時間來解決，
最後關頭才上傳了作業，才剛準備要鬆口氣，就是第一次期中考的到來！

轉變：對於三大基礎有了基礎的任務，了解哪些元件需要哪些技術來構築完成
> 比較難理解的是 callback function 以及 LocalStory 資料的進階應用

### 中期：(實際時間：後三週課程)
好不容易地吸收資訊，努力的完成且通過了第一次期中考，就準備來迎接 React 的實戰開發了，實在是很興奮啊！

主要學習：
1. 事件處理
2. 狀態管理
3. 命名的重要性

- hook 各自帶不同的變數（地方政府）
- redux 帶相同變數（統一集權）

轉變：開始了解並使用前端開發的思維，不斷嘗試錯誤，累積經驗延伸到下個技術
> 好不容易爬到了第二次期中考，差了些分數通過，這時才深知自己的不足之處，
> 花了些時間沈澱，開始思考要往哪個方向繼續前進。
## 後期 & 小結：（實際時間：過年後、第八週～現在）

感謝老師、助教、同學們。
讓我學到了好多從未學習過的，也在這次參加過程中，
建立了自信心，讓我能夠持續堅強地往前邁進，

利用前面兩個個階段所累積的經驗值，目前開始補足自身比較不足的領域，

真期待未來能夠再次參與這類型的計畫，並且成功達到最後，
在這做個心得紀錄，期許自己在未來能夠更進步。
